Part 1: Data Part 2: HTML

October 26th, 2020

Our final prompt was to create wireframes that illustrated potential user flows through an ideal version of our respective websites. While I did not return to my HTML project to update and design my codes, I was able to use Figma to delineate possible paths and patterns in a user flow, while signifying how I might allow users to correct mistakes.

I knew from my last project that I needed to find a better way to stack the plethora of information this project had uncovered. Thus, for my final designs, I focused on creating a simple prompt for users upon their logging on. Steps one and two in the process prompt the user with simple questions about the factors that matter most to them in property hunting. Only after these questions are answered is the user shown more detailed information about properties that might fit their needs. You can play with the prototype yourself in the framed site window below. (Yes, it works! Click to your heart's content.)

I found that by layering information, I could also layer menus on top of one another. This signified not only how deeply a user had gone into the site, but also allowed users to navigate anywhere on the site without backtracking.

One issue I did run into, though, was the inclusion of a “Search” bar so common in reference websites. Not being too familiar with AJAX, I wouldn’t know how exactly to implement such an element within my own webpage. However, this was an "idealized" version of my site, so I included it in my wireframes all the same.

Once I had created my Figma wireframes, I used Microsoft Word to integrate them into a user flow that would be easier to follow. By illustrating user pain points and mistakes and utilizing a word processor to discuss corrections and the website’s features, I wished for my user flow to be as easy to follow as possible. You can find that document below.

Though I had previous experience using Figma for UX Design projects, being able to actually develop wireframes helped me see not only how much I had to learn about web development, but also how much I had learned about data and information architecture.

Seeing my design in Figma helped me understand the significant gaps between designing a website and actually building a website. In addition, my work in this course helped inspire me to lay out this information in this way; simple taxonomies, metatagging, data analyses and learning to visualize data all aided me in the creation of this final product.

I attempted to maintain a similar method of organizing information in the development of this website, and will probably continue to use those ideas going forward. I also plan to learn about AJAX and how I can use in my future work. While this project turned out well, I'll definitely take the lesson of design vs. building with me; some ideas that look great in Figma simply can't be done in real life!
